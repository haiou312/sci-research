# Email Spec — Gmail SMTP Delivery of Daily News Reports

Loaded at the end of the Workflow (Step 10) when the user passes `--email`. Describes the auto-generated subject, body template, attachment selection, and failure handling for `scripts/send-report-email.py`.

## When This Step Runs

Only when the `email` parameter is non-empty. If `email` is empty or missing, skip this step silently — local files are still delivered per Step 8 and Step 9.

## Environment Variables Required

Claude's Bash environment must have these exported (typically via `~/.zshrc` or `~/.bash_profile`):

| Variable | Required | Default | Purpose |
|---|---|---|---|
| `GOOGLE_EMAIL_USERNAME` | Yes | — | Gmail address (e.g. `you@gmail.com`) |
| `GOOGLE_EMAIL_APP_PASSWORD` | Yes | — | 16-char Gmail App Password |
| `GOOGLE_EMAIL_FROM_NAME` | No | = `USERNAME` | Display name in From header |
| `GOOGLE_EMAIL_HOST` | No | `smtp.gmail.com` | SMTP host |
| `GOOGLE_EMAIL_PORT` | No | `587` | SMTP port |
| `GOOGLE_EMAIL_START_TLS` | No | `true` | STARTTLS on port 587 |

See `.env.example` at the repo root.

**Never log, echo, or print the app password.** The skill prompt must instruct the orchestrator not to read these variables into its own output; they flow only through the Python subprocess environment.

## Subject Template

Default (override via `--email-subject`):

```
{country_display} {title_label} — {date_display}
```

Rendered per `lang` Localisation Table:

- `lang=zh` → `中国每日热点新闻 — 2026年4月14日`
- `lang=en` → `China Daily News Intelligence — April 14, 2026`
- `lang=ja` → `中国デイリーニュース — 2026年4月14日`

## Attachment Selection

Controlled by `email_attach` parameter:

| Value | Behaviour |
|---|---|
| `both` (default) | Attach both `{out_md}` and `{out_docx}` |
| `docx` | Attach only `{out_docx}` |
| `md` | Attach only `{out_md}` |
| `none` | Send no attachments (body-only email) |

## Invocation

The skill orchestrator invokes the Python script via Bash. Write the body to a temp file first to avoid shell-quoting issues with multi-line content and Unicode quotes:

```bash
# Create temp body file
BODY_FILE=$(mktemp /tmp/daily-news-body.XXXXXX.txt)
cat > "$BODY_FILE" <<'EOF'
<body content here>
EOF

# Determine attachments based on email_attach parameter
case "$email_attach" in
  both) ATTACHMENTS=("$out_md" "$out_docx") ;;
  docx) ATTACHMENTS=("$out_docx") ;;
  md)   ATTACHMENTS=("$out_md") ;;
  none) ATTACHMENTS=() ;;
esac

# Call the sender
python3 "${CLAUDE_PLUGIN_ROOT}/scripts/send-report-email.py" \
  --to "$email" \
  --subject "$email_subject" \
  --body-file "$BODY_FILE" \
  ${ATTACHMENTS[@]+--attach "${ATTACHMENTS[@]}"}

SEND_EXIT=$?
rm -f "$BODY_FILE"
```

If `--email-dry-run` is set, append `--dry-run` to the Python call. The script will print a summary and exit 0 without connecting to SMTP.

## Exit Code Handling

| Python script exit | Orchestrator action |
|---|---|
| 0 | Print "✅ email sent to {N} recipient(s)" |
| 1 | Print "⚠️ email skipped: missing env vars {list}. Local files are preserved." — do NOT delete reports |
| 2 | Print "⚠️ email skipped: invalid recipient format." — do NOT delete reports |
| 3 | Print "⚠️ email failed: SMTP error {message}. Local files are preserved." — do NOT delete reports |
| 4 | Print "⚠️ email failed: attachment not found. Local files are preserved." — do NOT delete reports |
| 5 | Print "⚠️ email failed: body file not found." — do NOT delete reports |

**Hard rule**: email failure never deletes, moves, or overwrites the local `.md` or `.docx` files. The report has already been delivered by Step 8 and Step 9.

## Security

- Never `echo`, `cat`, or interpolate `$GOOGLE_EMAIL_APP_PASSWORD` into any command that prints to the terminal.
- The Python subprocess inherits env vars implicitly — do NOT pass the password via `--flag` argument (it would show in process lists).
- Do not write the password to the temp body file, log files, or any stdout.
- Validate recipient addresses with the regex `^[^@\s]+@[^@\s]+\.[^@\s]+$` before sending (the Python script does this; the orchestrator does not need to duplicate the check).
- If you plan to share logs, scrub any accidental leak of `GOOGLE_EMAIL_USERNAME` too.

## Testing

Recommended order for first-time setup:

1. Verify env vars are set (without leaking passwords):
   ```bash
   [ -n "$GOOGLE_EMAIL_USERNAME" ] && echo "username: set" || echo "username: MISSING"
   [ -n "$GOOGLE_EMAIL_APP_PASSWORD" ] && echo "password: set" || echo "password: MISSING"
   ```
2. Dry-run the full pipeline:
   ```
   /daily-news-intelligence --country China --email you@gmail.com --email-dry-run
   ```
3. Send to yourself for inspection:
   ```
   /daily-news-intelligence --country China --email you@gmail.com
   ```
4. Check Gmail inbox — confirm subject, body render, attachments open correctly.
