# Output Format Rules

These formatting rules MUST be followed for all research article output.

## Word Count

- **Hard limit**: 5000 words (for zh: ~7500 characters including punctuation)
- **Soft target**: 3500-4500 words for optimal depth-to-readability ratio
- If over limit, reduce in this order:
  1. Section 5 (Trends & Outlook) — cut speculative content first
  2. Section 3 (Entity Analysis) — reduce examples, keep core data
  3. Section 2 (Core Concepts) — shorten history timeline
  4. NEVER cut: Abstract, Comparison Table, References

## Reference Format

Use APA 7th Edition with categorization:

**Academic Papers:**
```
[N] Author, A. A., & Author, B. B. (Year). Title of article. *Journal Name*, Volume(Issue), Pages. https://doi.org/xxx
```

**Reports:**
```
[N] Organization. (Year). *Title of report*. Publisher. URL
```

**News Articles:**
```
[N] Author, A. (Year, Month Day). Title of article. *Publication Name*. URL
```

**Web Pages:**
```
[N] Author or Organization. (Year). Title of page. Site Name. URL
```

## Inline Citations

- Use numbered references: [1], [2], [3]
- Multiple sources for one claim: [1][3][7]
- Place citation immediately after the claim, before the period
- Every factual claim MUST have at least one citation

## Tables

- All comparison tables must include a "Source" column
- Use consistent units across all entities in a table
- Mark missing data as "N/A" with a footnote explaining why
- Right-align numerical data, left-align text

## Entity Balance

- Each entity in Section 3 should receive 300-600 words
- Deviation >30% between entities requires justification
- If data is sparse for an entity, state this explicitly rather than padding
